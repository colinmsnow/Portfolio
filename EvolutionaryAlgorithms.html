Evolutionary Algorithms for Airfoil Design

Evolution, Modelling

An evolutionary algorithm which generates symmetric airfoil shapes by trial and error

This project has its own site which can be found here and goes into much more detail about how it works

This algorithm creates airfoils, or cross sections of wings, by iteratively testing and improving a set of parameters and evaluating the results using computational fluid dynamics. Shown above is one result of the program showing an excellent pressure graph for very high lift and low drag.

But what is an evolutionary algorithm? It is a program which uses the principles of evolution to produce solutions to problems which may not have an analytical solution. While normal programs start with some input and some transformation to produce an output, an evolutionary algorithm starts with a random input and slowly evolves this input until it is closer to the desired outcome. The program starts by spinnign up a bunch of random instances of the thing it wants to create, then evaluates each one on how well it performs some task, and does a sort of "survival of the fittest" on them where it removes the ones which perform poorly and combine the successes to create "children" which are made up of the successful attributes of the parents. When this is performed over and over again, these children perform better and better on the given task, and eventually converge on what is close to the optimal solution to the problem. This type of solution is remarkable because it does not require any type of knowledge of how to create successful instances, but only a way to evaluate how good they are.

In this case, the model instances are functions which define the shape of airfoils. Starting with a random set, they are each evaluated using XFOIL, a command line computational fluid dynamics program, to determine which have the highest lift to drag ratio. From there, they evolve slowly until they approach the optimal solution for the given problem. In the case of this project this is important because the normal way of producing airfoils requires extensive wind tunnel testing, which is not practical to perform for every possible environmental condition. Our program can adapt to conditions that have never required an airfoil before, and give a good result in minutes instead of months.
