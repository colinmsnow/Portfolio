Scan Optimization by Probabilistic Analysis

Algorithms, Statistics

A probabilistic methodology for quantifying information gathered from 3D scans and potential applications in scan optimization.

Background

Object scanning technology is seeing an explosion in popularity and usefulness as many technologies such as 3D LIDAR and Photogrammetry have become increasingly cheap and reliable. All of these technologies effectively work by taking multiple scans of an object from different perspectives and using this data to reconstruct a best approximation of its 3D form. The algorithms that do this computation are extremely good but they can only do so much to account for a lack of information that comes from incomplete data from the initial scan.

Currently, most of this scanning is performed by human drone operators who attempt to capture enough data to construct the model and not miss any spots. Given the complexity of the task, any human pilot is likely to either underscan some areas which would require another flight to correct or capture an excess of data which wastes flight time and computational complexity in reconstruction.

In order to optimize this process we would like to capture the maximum amount of information about the system per scan, thus minimizing the amount of time and data we need to achieve some level of model fidelity. One simple way to achieve this would be to use a simple heuristic like the number of times a point has been scanned to assess our uncertainty about its position. While this is certainly better than nothing, it fails to fully capture the measurement uncertainty and thus can give us misleading results about our data accuracy. For example if you were to take six pictures of the same point from roughly the same place you may get six references about that point's position but you would know very little more than you knew after the first image about how much error that estimate contains.

In order to understand how much information we get from an image we must first create a model for the uncertainty of our measurements. Many SLAM algorithms rely on modeling sensor uncertainty using some form of gaussian noise but this assumption turns out to have disasterous consequences when applied to sensors like 3D LIDAR because their error is inherently variable. Consider a LIDAR which as some small distance and angular uncertainty. Modelling this uncertainty with gaussian noise would yield a cloud evenly distributed about the target point. However, this does not account for the fact that horizontal position error is actually a function of distance since small angular errors grow into large linear errors at long distances. In the graphic below you can see that as points move further from the scanner their uncerainty grows along a circle centered at the scanner.

Once we have a suitable model which accounts for these non-gaussian errors, we can apply it to determine how a set of points should be observed in order to prodice the maximum amount of information per observation. Since our uncertainty model is likely to be too complex to calculate analytically, we use Monte Carlo sampling to create expected distributions around all of our observed points. This then lets us calculate measures like the standard deviation of these samples which gives us a good heuristic for uncertainty. Then, when we are deciding where to take the next sample, we can again calculate expected distributions about our estimate of the object's position and perform gradient descent to find the point which minimizes our uncertainty about the position of all the points, thus maximizing the information gained about the system.

The outputs of this model could then suggest next points for pilots to fly to and tell them when they have captured enough information to create a model to a certain tolerance. Alternatively, the model could be adapted to calculate routes for autonomous vehicles such that they capture the maximum amount of information over the route as a whole.
