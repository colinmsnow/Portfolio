Cable Bot

Underwater, Environmental Sensing

A robotic platform to automate coastal monitoring

Cable Bot is a robot which is designed to carry environmental sensor packages and deploy them repeatably without human intervention. The system consists of a cable-traversing robot attached to a fixed wire which can move sensor packages into and out of the water at set positions completely atonomously. This allows for long term studies in remote locations, even in inclement weather, to get data which is critical to understanding the oceans.

We worked on this project for ten weeks and deployed a finished first prototype in Buzzards Bay in August 2020. Footage from one run is seen below. The water was very opaque so it is sometimes hard to see what is happening, but the robot moves down the cable one meter at a time taking measurements each time it stops, then returns to the surface.

For a deeper explanation about the application of this technology please read our abstract for this project.

Construction

The robot consists of two main subsystems: The motorized upper half and the lower electronics tube. The top assembly is responsible for moving the robot up and down the cable while the bottom assembly holds the electronics which handle decision making.

Top Assembly

The top assembly consists of a dc motor which spins two wheels that make contact with the cable. The wheels are driven by a belt system which is covered by a waterproof casing to protect it from the surrounding water. The only moving pieces that extend out of the waterproof section are the two driven shafts, which are each sealed by a spring-loaded shaft seal. Because the rest of the assembly does not contain any sensitive elctronics, it does not need to be waterproofed, dramatically simplifying the design.

Outside of this enclosure two sets of wheels clamp down on the cable to get the required grip needed to move the robot through rough weather. The driven set on the top have a helical pattern cut into them to match the shape of the cable, dramatically increasing purchase. The clamping action is achieved with two hand latches on sliders, allowing the robot to be put on and taken off the cable in a matter of seconds while also allowing adjustment of clamping force.

Bottom Assembly

The bottom assembly houses all of the electronics which handle motor control, decision making, and communications. Its structure is very simple, consisting of a 4 inch pvc tube with a double o-ring seal on the back end. The front side has a polycarbonate window so that the onboard camera can see the water and the back side has an aluminum plate through which all the wires are run.

Electronics

The autonomous nature of this robot necessitates a lot of onboard computation and decision making as well as robust communication. To achieve this we employed a number of different electronic systems, each of which connect to the others over serial lines. The main brain of the system is a Raspberry Pi model 4 which holds all of the subsystems together while also handling external communication. This connects to a Teensy 4.1 which handles motor control and general robot health and the sensor package (in this case an InSitu Aqua Troll 500) which does all the environmental tests. All of this is packaged in the main electronics tube (the yellow one) along with a motor controller, temperature sensor, and a system of sensors to detect leaks. There is also a front facing camera and emergency stop sensor just behind the front window.

Code

All code from this project can be found on GitHub here.

The software for this system needs to handle a lot of different things at once, and is therefore quite complicated as a whole. This complexity is broken down by allowing each of the distinct functions of the system to run in a separate thread and communicate through a series of pipes and queues, thus reducing the dependencies between them to an absolute minimum. These processes are:

  • Main web app to interface with user
  • Deployment manager to coordinate autonomous motion
  • Modbus communication with sensors
  • Motor and sensor control with Teensy
  • Front camera control and data management
  • SQL database manager

Interface

To make controlling the robot as easy as possible, we created a web interface which allows a user to look at real time data, control the robot, and write deployment scripts to be run autonomously. The interface is hosted on a local wifi network on the Pi and is accessible whenever the robot is at the surface (wifi gets attenuated quickly underwater). When the robot submerges the page is no longer accessible, but the robot continues to run pre-scripted commands until it is accessed again.
